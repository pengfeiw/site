# AGENTS.md - Coding Guidelines for Site Repository

## Project Overview

This is a **static personal blog website** hosted on GitHub Pages at `pengfeixc.com`. It consists of pure HTML, CSS, and JavaScript files with **no build system, no package manager, and no test framework**.

**Content Architecture**:
- Landing page (`index.html`) with Nier: Automata-inspired terminal UI
- Blog section (`/blogs/`) with 11 topic categories
- Content appears to be generated by an external tool (site-generator)
- Chinese language content (zh_cn locale)

## Commands

**No build, lint, or test commands exist.**

This is a static website - files are served as-is:
- No `npm run build` - no build step required
- No `npm test` - no test framework configured
- No `npm run lint` - no linting configured
- No `npm install` - no package.json exists

**Manual verification**:
```bash
# Open files directly in browser for testing
# Or use a simple HTTP server:
python -m http.server 8000
npx serve .
```

## Code Style Guidelines

### HTML

**Structure**:
- Use HTML5 doctype: `<!DOCTYPE html>`
- Set language attribute: `<html lang="zh_cn">`
- Include viewport meta tag for responsive design
- Use semantic HTML elements (header, nav, main, article, footer)

**Formatting**:
- **Blog pages**: Minified/single-line HTML output is acceptable (content is generated)
- **Hand-written pages** (index.html, 404.html): Use readable formatting with proper indentation (2 spaces)
- Include descriptive meta tags for SEO

**Patterns**:
```html
<!-- External resources -->
<link rel="stylesheet" href="/blogs/assests/global.css">
<script type="text/javascript" src="..."></script>

<!-- Icons -->
<link rel="icon" href="/blogs/assests/images/icon.jpg">
```

### CSS

**Architecture**:
- Modular CSS files in `/blogs/assests/`:
  - `global.css` - Base styles and font definitions
  - `article.css` - Article layout with TOC sidebar
  - `index.css` - Blog homepage styles
  - `navigator.css` - Navigation header
  - `github_markdown.css` - GitHub-style markdown rendering
  - `code_github.css` - Syntax highlighting

**Naming Conventions**:
- Use kebab-case for class names: `.article-item`, `.category-list`
- Use semantic class names describing purpose, not appearance
- Prefix component classes: `.nier-btn`, `.article-list`

**CSS Variables**:
```css
:root {
  --bg-color: #dcd8c0;
  --text-color: #4e4b42;
  --accent-color: #bab5a1;
  --highlight: #c13d3d;
  --border-color: #5c594d;
}
```

**Box Model**:
```css
*, :before, :after {
  box-sizing: border-box;
}
```

**Responsive Design**:
- Use CSS Grid and Flexbox for layouts
- Include mobile breakpoints with `@media (max-width: 480px)`

### JavaScript

**Style**:
- Use vanilla JavaScript only (no frameworks)
- ES5/ES6 syntax acceptable
- Add event listeners after DOM is ready:

```javascript
window.addEventListener('DOMContentLoaded', function() {
  // Initialization code
});
```

**Patterns**:
```javascript
// Element selection
document.getElementById('search-input')

// Event delegation for dynamic content
 element.addEventListener('search', (event) => {
   // Handle event
 });
```

### Accessibility & SEO

- Include proper meta description tags
- Use alt text for images
- Maintain semantic HTML structure
- Provide RSS feed at `/blogs/rss.xml`
- Include XML sitemap at `/blogs/blogSitemap.xml`

### Error Handling

Since this is a static site with no backend:
- Use `404.html` for client-side error pages
- Validate external links manually
- Test in multiple browsers (Chrome, Firefox, Safari, Edge)
